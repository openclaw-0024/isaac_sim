## 阶段 1：Isaac Lab 基础概念（3天）

### 课程地址

https://docs.nvidia.com/learning/physical-ai/getting-started-with-isaac-lab/latest/index.html

### 学习内容

1. **Physical AI 概念**
   - 什么是 Physical AI
   - 机器人为什么需要在仿真中学习
   - 并行仿真的优势

2. **强化学习核心原理**
   - 马尔可夫决策过程 (MDP)
   - 策略 (Policy)、奖励 (Reward)、价值函数 (Value Function)
   - 探索与利用 (Exploration vs Exploitation)

3. **Isaac Lab 架构理解**
   - **Scene**: 仿真场景，包含机器人、地形、物体
   - **Environment**: RL 环境封装，管理观测/动作/奖励
   - **Manager**: 模块化管理器（ObservationManager, RewardManager, ActionManager 等）
   - **Config**: 通过 Python dataclass 进行配置

- [ ] 理解 `ManagerBasedRLEnv` 和 `DirectRLEnv` 的区别

## 阶段 2：CartPole 入门训练（3天）

### 课程地址

https://docs.nvidia.com/learning/physical-ai/getting-started-with-isaac-lab/latest/train-your-first-robot-with-isaac-lab/00-index.html

### 实操步骤

#### 步骤 1：了解 CartPole 环境

CartPole 是最经典的 RL 入门环境：一根杆子插在小车上，通过左右移动小车保持杆子直立。

```bash
# 先可视化看看 CartPole 长什么样
./isaaclab.sh -p source/isaaclab_tasks/isaaclab_tasks/manager_based/classic/cartpole/cartpole_env_cfg.py
```

#### 步骤 2：理解环境配置

关键文件结构：
```
isaaclab_tasks/manager_based/classic/cartpole/
├── __init__.py              # 注册环境
├── cartpole_env_cfg.py      # 环境配置（场景、观测、奖励、动作）
```

重点理解：
- **Observation**: 小车位置、小车速度、杆子角度、杆子角速度
- **Action**: 小车左右推力
- **Reward**: 杆子越直立奖励越高，倒了就结束
- **Termination**: 杆子倾斜超过阈值 或 小车超出边界

#### 步骤 3：训练

```bash
./isaaclab.sh -p scripts/reinforcement_learning/rsl_rl/train.py \
  --task Isaac-CartPole-v0 \
  --num_envs 1024 \
  --headless
```

#### 步骤 4：评估训练结果

```bash
./isaaclab.sh -p scripts/reinforcement_learning/rsl_rl/play.py \
  --task Isaac-CartPole-v0 \
  --num_envs 16
```

#### 步骤 5：创建外部项目（扩展练习）

课程地址：https://docs.nvidia.com/learning/physical-ai/getting-started-with-isaac-lab/latest/train-your-first-robot-with-isaac-lab/05-create-and-install-a-new-external-project.html

## 阶段 3：理解环境自定义（3天）

### 学习内容

通过修改 CartPole 环境来理解自定义流程：

1. **修改奖励函数** — 尝试改变奖励权重，观察训练行为变化
2. **修改观测空间** — 增加或减少观测量
3. **修改动作空间** — 调整动作范围
4. **修改终止条件** — 调整什么情况下 episode 结束

### 实操练习

```python
# 尝试修改 cartpole_env_cfg.py 中的奖励权重
# 例如：增大角度惩罚，减小位置惩罚
# 观察训练结果有什么不同
```

- [ ] 理解不同奖励权重对训练结果的影响

## 阶段 4：四足机器人 Locomotion（5天）

### 参考资源

- NVIDIA Blog: https://developer.nvidia.com/blog/train-a-quadruped-locomotion-policy-and-simulate-cloth-manipulation-with-nvidia-isaac-lab-and-newton/
- Isaac Lab 内置 ANYmal 环境

### 实操步骤

#### 步骤 1：运行预训练 demo

```bash
# ANYmal 四足机器人在平坦地形
./isaaclab.sh -p scripts/reinforcement_learning/rsl_rl/play.py \
  --task Isaac-Velocity-Flat-Anymal-D-v0 \
  --num_envs 32

# ANYmal 在崎岖地形
./isaaclab.sh -p scripts/reinforcement_learning/rsl_rl/play.py \
  --task Isaac-Velocity-Rough-Anymal-D-v0 \
  --num_envs 32
```

#### 步骤 2：训练四足 locomotion

```bash
./isaaclab.sh -p scripts/reinforcement_learning/rsl_rl/train.py \
  --task Isaac-Velocity-Flat-Anymal-D-v0 \
  --num_envs 4096 \
  --headless
```

#### 步骤 3：理解 Velocity Tracking 任务框架

这是 locomotion 任务的核心框架，后续人形机器人也用同样的结构：

- **命令 (Command)**: 目标线速度 (vx, vy) + 目标角速度 (wz)
- **观测 (Observation)**: 基座姿态、关节状态、上一步动作、命令
- **动作 (Action)**: 各关节位置目标
- **奖励 (Reward)**: 速度跟踪 + 各种正则化惩罚项

#### 步骤 4：理解 Terrain Curriculum

```
Level 0: 平坦地形 (flat)
Level 1: 缓坡 (slopes)
Level 2: 随机凸起 (random uniform)
Level 3: 台阶 (stairs)
Level 4: 离散障碍 (discrete obstacles)
```

机器人从简单地形开始，表现好了自动升级到更难的地形。

### 验证

- [ ] 理解 velocity tracking 奖励函数的每一项
- [ ] 理解 terrain curriculum 的工作原理
