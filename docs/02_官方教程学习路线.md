### 学习内容

1. **Physical AI 概念**
   - 什么是 Physical AI
   - 机器人为什么需要在仿真中学习
   - 并行仿真的优势

2. **强化学习核心原理**
   - 马尔可夫决策过程 (MDP)
   - 策略 (Policy)、奖励 (Reward)、价值函数 (Value Function)
   - 探索与利用 (Exploration vs Exploitation)

3. **Isaac Lab 架构理解**
   - **Scene**: 仿真场景，包含机器人、地形、物体
   - **Environment**: RL 环境封装，管理观测/动作/奖励
   - **Manager**: 模块化管理器（ObservationManager, RewardManager, ActionManager 等）
   - **Config**: 通过 Python dataclass 进行配置

#### 步骤 2：理解环境配置

源码路径（相对 Isaac Lab 仓库根目录）：

```
source/isaaclab_tasks/isaaclab_tasks/manager_based/classic/cartpole/
├── __init__.py              # 注册 Gym 任务（如 Isaac-Cartpole-v0）
├── cartpole_env_cfg.py      # 本环境的核心配置（下列各类 Cfg）
├── mdp/                     # 观测/奖励/终止等具体函数实现
```

机器人资产引用自 `isaaclab_assets/robots/cartpole.py`（USD 中的关节名：`slider_to_cart` 小车滑轨，`cart_to_pole` 摆杆）。

##### `CartpoleSceneCfg`（场景）

| 字段 | 含义 |
|------|------|
| `ground` | 100×100 地面，prim `/World/ground` |
| `robot` | `CARTPOLE_CFG.replace(prim_path="{ENV_REGEX_NS}/Robot")`，多环境克隆时每个 env 单独一份 |
| `dome_light` | 穹顶光；`color=(0.9,0.9,0.9)`，`intensity=500` |

##### `ActionsCfg`（动作）

- `joint_effort`：`JointEffortActionCfg`，只对 **`slider_to_cart`** 施加力矩类动作（摆杆关节在资产里以被动/阻尼为主，不由策略直接驱动）。
- `scale=100.0`：策略输出乘该系数后再作为关节力，用于调节动作幅度。

##### `ObservationsCfg`（观测）

- `policy` 组内：`joint_pos_rel`、`joint_vel_rel`（相对默认姿态的关节位置与速度；两关节都会进入观测，顺序由 `mdp` 实现决定）。
- `enable_corruption = False`：本组观测不加随机噪声/故障。
- `concatenate_terms = True`：多条 term 拼成一向量给策略。

##### `EventCfg`（事件，主要在 reset）

- `reset_cart_position`：`slider_to_cart` 在 reset 时位置扰动约 **[-1, 1]**，速度 **[-0.5, 0.5]**（相对默认的随机偏移，具体见 `mdp.reset_joints_by_offset`）。
- `reset_pole_position`：`cart_to_pole` 角度 **±0.25π rad**，角速度 **±0.25π**，增加起跑随机性。

##### `RewardsCfg`（奖励，加权求和）

| 项 | weight | 作用简述 |
|----|--------|----------|
| `alive` | `+1.0` | 存活每步加分 |
| `terminating` | `-2.0` | 触发终止时的惩罚 |
| `pole_pos` | `-1.0` | `joint_pos_target_l2`：摆杆关节偏离目标 `0`（竖直）越远代价越大 |
| `cart_vel` | `-0.01` | 小车关节速度 L1，抑制猛冲 |
| `pole_vel` | `-0.005` | 摆杆角速度 L1，抑制剧烈晃动 |

##### `TerminationsCfg`（终止）

本文件内**显式**配置的两项为：

- `time_out`：`episode_length_s` 耗尽（配合下方 `CartpoleEnvCfg` 的 `episode_length_s`）；`time_out=True` 供算法区分截断。
- `cart_out_of_bounds`：`slider_to_cart` 超出 **[-3.0, 3.0]** 结束 episode。

> 配置里未单独写「杆倒角超过某阈值」的 termination；倾倒主要由动力学 + 奖励塑形体现。若需硬终止，要在本类或 `mdp` 中自行添加 term。

##### `CartpoleEnvCfg`（整环境）

| 参数 | 含义 |
|------|------|
| `scene` | 默认 `num_envs=4096`，`env_spacing=4.0`，`clone_in_fabric=True`（大规模并行） |
| `decimation` | `2`：每 **2** 个物理子步才对应 1 次 RL `step` |
| `episode_length_s` | `5` 秒 |
| `sim.dt` | `1/120` s → 物理 **120 Hz**；配合 `decimation=2`，控制约 **60 Hz** |
| `sim.render_interval` | 等于 `decimation`，与 RL 步对齐 |
| `viewer.eye` | `(8.0, 0.0, 5.0)` 默认相机 |

### 学习内容

1. **修改奖励函数** — 尝试改变奖励权重，观察训练行为变化

4. **修改终止条件** — 调整什么情况下 episode 结束

#### 步骤 3：理解 Velocity Tracking 任务框架

这是 locomotion 任务的核心框架，后续人形机器人也用同样的结构：

- **命令 (Command)**: 目标线速度 (vx, vy) + 目标角速度 (wz)
- **观测 (Observation)**: 基座姿态、关节状态、上一步动作、命令
- **动作 (Action)**: 各关节位置目标
- **奖励 (Reward)**: 速度跟踪 + 各种正则化惩罚项

#### 步骤 4：理解 Terrain Curriculum

```
Level 0: 平坦地形 (flat)
Level 1: 缓坡 (slopes)
Level 2: 随机凸起 (random uniform)
Level 3: 台阶 (stairs)
Level 4: 离散障碍 (discrete obstacles)
```

机器人从简单地形开始，表现好了自动升级到更难的地形。
