## 1. 可用的宇树人形机器人环境

Isaac Lab 内置了宇树 (Unitree) H1 和 G1 两款人形机器人的 locomotion 任务：

| 任务 ID | 机器人 | 地形 | 描述 |
|---------|--------|------|------|
| `Isaac-Velocity-Flat-H1-v0` | H1 | 平坦 | 入门首选，最简单的配置 |
| `Isaac-Velocity-Rough-H1-v0` | H1 | 崎岖 | 带地形 curriculum |
| `Isaac-Velocity-Flat-G1-v0` | G1 | 平坦 | G1 体型更小，控制不同 |
| `Isaac-Velocity-Rough-G1-v0` | G1 | 崎岖 | G1 崎岖地形 |

### H1 vs G1 对比

| 特性 | Unitree H1 | Unitree G1 |
|------|-----------|-----------|
| 身高 | ~1.8m | ~1.3m |
| 体重 | ~47kg | ~35kg |
| 自由度 | 19 DOF | 23-29 DOF（视版本） |
| 特点 | 结构简单，适合入门 | 更灵活，上肢自由度多 |
| 推荐顺序 | 先学 | 后学 |

## 2. 第一步：运行 H1 平坦地形（入门）

### 2.1 训练

```bash
# 进入 IsaacLab 目录
cd /path/to/IsaacLab

# 训练 H1 在平坦地形行走
# --num_envs: 并行环境数（根据 GPU 显存调整）
# --headless: 无 GUI，加快训练速度
./isaaclab.sh -p scripts/reinforcement_learning/rsl_rl/train.py \
  --task Isaac-Velocity-Flat-H1-v0 \
  --num_envs 4096 \
  --headless

# 如果 GPU 显存不够 4096 个环境，可以减少：
# RTX 3080 16GB: --num_envs 2048
# RTX 4090 24GB: --num_envs 4096
# A100 80GB:     --num_envs 8192
```

训练输出会保存在 `logs/rsl_rl/` 目录下。

### 2.2 监控训练

训练过程中会输出类似信息：

```
[INFO] Iteration: 100/1500
  Mean reward:      12.34
  Mean episode len:  450
  Policy loss:       0.012
  Value loss:        0.034
```

关注指标：
- **Mean reward**: 应该持续增长，最终趋于稳定
- **Mean episode length**: 应该越来越长（机器人存活越久）
- **Policy loss / Value loss**: 应该逐渐减小

### 2.3 可视化评估

```bash
# 训练完成后，回放策略
./isaaclab.sh -p scripts/reinforcement_learning/rsl_rl/play.py \
  --task Isaac-Velocity-Flat-H1-v0 \
  --num_envs 50

# 指定 checkpoint（如果有多个）
./isaaclab.sh -p scripts/reinforcement_learning/rsl_rl/play.py \
  --task Isaac-Velocity-Flat-H1-v0 \
  --num_envs 50 \
  --load_run <run_name>
```

### 2.4 预期结果

训练约 1000-1500 iterations 后：
- H1 能在平坦地形上稳定行走
- 能跟踪给定的速度命令（前进、转弯）
- 步态自然，无明显抖动

### 3.2 理解 flat_env_cfg.py（H1 平坦地形）

```python
# 这是简化版本，帮助理解结构

class H1FlatEnvCfg(LocomotionVelocityRoughEnvCfg):
    """H1 平坦地形环境配置，继承自崎岖地形基类并做简化"""

    def __post_init__(self):
        super().__post_init__()

        # --- 场景配置 ---
        # 平坦地形，无高度扫描
        self.scene.terrain.terrain_type = "plane"
        self.scene.height_scanner = None
        self.observations.policy.height_scan = None

        # --- 奖励调整 ---
        # 脚部空中时间奖励（鼓励迈步而不是拖地）
        self.rewards.feet_air_time.weight = 1.0
        self.rewards.feet_air_time.params["threshold"] = 0.6

        # --- 评估模式配置 ---
        # 用于 play 时的精简配置
        # 50 个环境、禁用观测噪声、去掉随机扰动
```

### 3.3 理解基础环境 velocity_env_cfg.py

这是所有 locomotion velocity tracking 环境的父类，核心组件：

#### 观测空间 (Observations)

```python
# 典型观测向量包含：
observations = {
    "base_lin_vel":      3,   # 基座线速度 (x, y, z) in body frame
    "base_ang_vel":      3,   # 基座角速度 (roll, pitch, yaw rate)
    "projected_gravity":  3,   # 重力在body frame的投影
    "velocity_commands":  3,   # 速度命令 (vx, vy, wz)
    "joint_pos":         N,   # 各关节位置（相对默认位置的偏差）
    "joint_vel":         N,   # 各关节速度
    "actions":           N,   # 上一步的动作
}
# H1 的 N = 19 (19个可动关节)
```

#### 动作空间 (Actions)

```python
# 动作 = 各关节的位置目标偏移量
# action_dim = 关节数 (H1: 19)
# 实际发给关节的目标 = default_joint_pos + action * action_scale
```

#### 奖励函数 (Rewards)

```python
rewards = {
    # --- 正向奖励（鼓励） ---
    "track_lin_vel_xy_exp":  1.0,    # 跟踪 xy 线速度
    "track_ang_vel_z_exp":   0.5,    # 跟踪 z 角速度
    "feet_air_time":         1.0,    # 脚离地时间（鼓励迈步）

    # --- 惩罚项（抑制不良行为） ---
    "lin_vel_z_l2":         -2.0,    # z方向线速度（抑制弹跳）
    "ang_vel_xy_l2":        -0.05,   # xy角速度（抑制摇晃）
    "dof_torques_l2":       -1.0e-5, # 关节力矩（抑制暴力）
    "dof_acc_l2":           -2.5e-7, # 关节加速度（抑制抖动）
    "action_rate_l2":       -0.01,   # 动作变化率（抑制突变）
    "undesired_contacts":   -1.0,    # 非预期接触（膝盖触地等）
    "flat_orientation_l2":  -1.0,    # 基座倾斜（保持直立）
}
```

### 3.4 关键参数解读

| 参数 | 含义 | H1 典型值 |
|------|------|----------|
| `physics_dt` | 物理仿真步长 | 1/200s (0.005s) |
| `control_dt` | 控制频率（决策间隔） | 1/50s (0.02s) |
| `decimation` | physics_dt 与 control_dt 的比值 | 4 |
| `episode_length_s` | 单个 episode 最大时长 | 20s |
| `num_envs` | 并行环境数量 | 4096 |
| `env_spacing` | 各环境之间的间距 | 2.5m |


### Flat vs Rough 差异

| 差异点 | Flat | Rough |
|--------|------|-------|
| 地形 | 平面 | 多种地形（斜坡、台阶、随机凸起） |
| 高度扫描 | 无 | 有（用于感知地形） |
| 观测维度 | 较小 | 较大（增加高度图） |
| 训练难度 | 低 | 高 |
| 训练时间 | ~1000 iterations | ~2000-3000 iterations |
| 泛化能力 | 差（只适应平面） | 强（适应多种地形） |

### G1 与 H1 的训练差异

- G1 自由度更多，动作空间更大，训练可能需要更多迭代
- G1 体型更小更轻，平衡特性不同
- G1 官方 USD 模型与 Isaac Lab 内置模型可能有差异（腰部位置、DOF 数）
- 如果使用官方 Unitree G1 USD 模型，需要重新调参

## 7. 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 机器人训练时一直摔倒 | 奖励函数不合理 | 检查 reward 权重，增大存活奖励 |
| 训练不收敛 | 学习率过大/过小 | 调整 PPO 超参数 |
| 机器人原地打转 | 角速度奖励过大 | 减小 `track_ang_vel_z` 权重 |
| 步态不自然（滑步） | 脚接触检测问题 | 检查 feet_air_time 配置 |
| 显存不足 | num_envs 过大 | 减少 num_envs |
