## 1. 准备机器人模型

### 1.1 URDF 文件要求

URDF (Unified Robot Description Format) 是描述机器人模型的标准格式。你的 URDF 必须包含：

```xml
<?xml version="1.0"?>
<robot name="my_humanoid">
  <!-- 基座 link（骨盆/躯干） -->
  <link name="base_link">
    <visual>
      <geometry><mesh filename="meshes/base.obj"/></geometry>
    </visual>
    <collision>
      <geometry><mesh filename="meshes/base_collision.obj"/></geometry>
    </collision>
    <inertial>
      <mass value="10.0"/>
      <inertia ixx="0.1" iyy="0.1" izz="0.05" ixy="0" ixz="0" iyz="0"/>
    </inertial>
  </link>

  <!-- 关节定义 -->
  <joint name="left_hip_yaw" type="revolute">
    <parent link="base_link"/>
    <child link="left_thigh"/>
    <axis xyz="0 0 1"/>
    <limit lower="-1.57" upper="1.57" effort="100" velocity="10"/>
  </joint>

  <!-- ... 其他 link 和 joint ... -->
</robot>
```

### 1.2 URDF 精度核查清单

| 检查项 | 说明 | 重要程度 |
|--------|------|---------|
| 质量 (mass) | 每个 link 的质量必须与实际硬件一致 | 极高 |
| 惯性矩阵 (inertia) | 从 CAD 模型导出，不能随便填 | 极高 |
| 质心位置 (origin) | inertial frame 的 origin 必须准确 | 极高 |
| 关节限位 (limit) | lower/upper 匹配实际电机行程 | 高 |
| 关节力矩限制 (effort) | 匹配电机峰值力矩 | 高 |
| 关节速度限制 (velocity) | 匹配电机最大角速度 | 中 |
| 碰撞模型 (collision) | 使用简化凸包，不用原始 mesh | 高 |
| 可视化模型 (visual) | 可用精细 mesh，仅影响显示 | 低 |

### 1.3 常见 URDF 来源

- **SolidWorks → URDF**: 使用 `sw_urdf_exporter` 插件
- **Fusion 360 → URDF**: 使用 `fusion2urdf` 插件
- **OnShape → URDF**: 使用 `onshape-to-robot` 工具
- **手写 URDF**: 适合简单模型或微调

### 1.4 碰撞模型简化

```
原始 mesh (数万三角面) → 凸分解 → 简化凸包 (数百三角面)
```

工具推荐：
- **V-HACD**: 凸分解算法（Isaac Sim 内置支持）
- **MeshLab**: 手动简化 mesh
- **trimesh** (Python): 编程方式处理 mesh

## 2. URDF 转 USD

### 2.1 Python 脚本方式（推荐，可批量处理）

```python
"""convert_urdf_to_usd.py - URDF 转 USD 转换脚本"""

from isaaclab.sim.converters import UrdfConverterCfg, UrdfConverter

cfg = UrdfConverterCfg(
    asset_path="/path/to/my_humanoid.urdf",
    usd_dir="/path/to/output/",
    usd_file_name="my_humanoid.usd",
    fix_base=False,           # 人形机器人 = 浮动基座，必须 False
    make_instanceable=True,   # 高效内存使用，并行环境必须 True
    merge_fixed_joints=True,  # 合并固定关节减少计算量
)
converter = UrdfConverter(cfg)
print(f"USD saved to: {converter.usd_path}")
```

### 2.2 Isaac Sim GUI 方式

1. 打开 Isaac Sim
2. `File` → `Import`
3. 选择 `.urdf` 文件
4. 配置选项：
   - **Import As**: `Robot`
   - **Fix Base Link**: `False`（人形机器人）
   - **Joint Drive Type**: `Position`（位置控制）或 `Effort`（力矩控制）
   - **Self Collision**: `Enable`
5. 点击 Import

### 2.3 验证 USD 模型

在 Isaac Sim 中打开生成的 USD 文件：
- 检查关节是否能正常旋转
- 检查碰撞体是否正确包裹 mesh
- 检查质心位置是否合理（使用 Physics Inspector）
- 让模型自由落体，看是否物理行为正常

```python
"""my_humanoid_cfg.py - 机器人资产配置"""

import isaaclab.sim as sim_utils
from isaaclab.actuators import ImplicitActuatorCfg
from isaaclab.assets.articulation import ArticulationCfg

MY_HUMANOID_CFG = ArticulationCfg(
    spawn=sim_utils.UsdFileCfg(
        usd_path="/path/to/my_humanoid.usd",
        rigid_props=sim_utils.RigidBodyPropertiesCfg(
            disable_gravity=False,
            retain_accelerations=False,
            max_depenetration_velocity=10.0,
        ),
        articulation_props=sim_utils.ArticulationRootPropertiesCfg(
            enabled_self_collisions=True,
            solver_position_iteration_count=4,
            solver_velocity_iteration_count=4,
        ),
    ),
    init_state=ArticulationCfg.InitialStateCfg(
        pos=(0.0, 0.0, 1.05),   # 初始高度（根据你的机器人身高调整）
        joint_pos={
            # 设置各关节的初始位置（站立姿态）
            "left_hip_pitch": -0.2,
            "left_knee": 0.4,
            "left_ankle_pitch": -0.2,
            "right_hip_pitch": -0.2,
            "right_knee": 0.4,
            "right_ankle_pitch": -0.2,
            # ... 其他关节
        },
    ),
    actuators={
        "legs": ImplicitActuatorCfg(
            joint_names_expr=[".*hip.*", ".*knee.*", ".*ankle.*"],
            stiffness=150.0,     # PD 控制刚度（需匹配实际电机）
            damping=5.0,         # PD 控制阻尼
            effort_limit=100.0,  # 力矩限制
        ),
        "torso": ImplicitActuatorCfg(
            joint_names_expr=[".*torso.*", ".*waist.*"],
            stiffness=100.0,
            damping=5.0,
            effort_limit=50.0,
        ),
    },
)
```

```python
from isaaclab_tasks.manager_based.locomotion.velocity.velocity_env_cfg import (
    LocomotionVelocityRoughEnvCfg,
)
from .my_humanoid_cfg import MY_HUMANOID_CFG

class MyHumanoidFlatEnvCfg(LocomotionVelocityRoughEnvCfg):
    def __post_init__(self):
        super().__post_init__()

        # 使用自定义机器人
        self.scene.robot = MY_HUMANOID_CFG.replace(prim_path="{ENV_REGEX_NS}/Robot")

        # 平坦地形
        self.scene.terrain.terrain_type = "plane"
        self.scene.height_scanner = None
        self.observations.policy.height_scan = None

        # 调整奖励权重（需要根据你的机器人特性调整）
        self.rewards.feet_air_time.weight = 1.0
        self.rewards.feet_air_time.params["threshold"] = 0.5

        # 指定脚部 body 名称（用于接触检测）
        self.rewards.feet_air_time.params["sensor_cfg"].body_names = [
            "left_foot", "right_foot"  # 替换为你的机器人脚部 link 名

        self.rewards.undesired_contacts.params["sensor_cfg"].body_names = [
            "left_thigh", "right_thigh", "torso"  # 不希望接触地面的部位
        ]
```

## 4. Reward Function 设计

### 4.1 人形机器人奖励函数设计原则

核心理念：**奖励你想要的行为，惩罚你不想要的行为**

#### 正向奖励（鼓励行为）

| 奖励项 | 权重范围 | 作用 |
|--------|---------|------|
| `track_lin_vel_xy` | 1.0 ~ 1.5 | 跟踪目标前进/侧移速度 |
| `track_ang_vel_z` | 0.5 ~ 0.75 | 跟踪目标转弯角速度 |
| `feet_air_time` | 0.5 ~ 1.0 | 鼓励脚交替离地（迈步） |
| `alive` | 0.1 ~ 0.5 | 存活奖励（不摔倒） |

#### 惩罚项（抑制行为）

| 惩罚项 | 权重范围 | 作用 |
|--------|---------|------|
| `lin_vel_z` | -1.0 ~ -2.0 | 抑制上下弹跳 |
| `ang_vel_xy` | -0.01 ~ -0.1 | 抑制左右摇晃 |
| `joint_torques` | -1e-5 ~ -1e-4 | 抑制暴力动作 |
| `joint_acc` | -1e-7 ~ -1e-6 | 抑制关节抖动 |
| `action_rate` | -0.005 ~ -0.02 | 抑制动作突变 |
| `flat_orientation` | -0.5 ~ -2.0 | 保持躯干直立 |
| `undesired_contacts` | -0.5 ~ -1.0 | 惩罚非脚部位触地 |
| `base_height` | -0.5 ~ -1.0 | 保持站立高度 |

### 4.2 调参策略

**阶段 1：先让机器人站起来**
```
track_lin_vel = 0     (先不追求速度)
alive = 1.0           (大力鼓励存活)
flat_orientation = -2.0 (强制直立)
base_height = -1.0    (保持站立)
```

**阶段 2：让机器人迈步**
```
track_lin_vel = 0.5   (开始追求速度)
feet_air_time = 1.0   (鼓励迈步)
alive = 0.5           (降低存活奖励比重)
```

**阶段 3：精调步态质量**
```
track_lin_vel = 1.0   (追求准确跟踪)
action_rate = -0.01   (抑制动作抖动)
joint_torques = -1e-5 (减少能量消耗)
joint_acc = -2.5e-7   (平滑关节运动)
```

## 5. 需要掌握的核心算法

### 5.1 PPO (Proximal Policy Optimization)

Isaac Lab locomotion 训练的标准算法，通过 RSL-RL 库实现。

**核心思想**：限制每次策略更新的幅度，防止训练不稳定。

**关键超参数**：

| 参数 | 说明 | 典型值 |
|------|------|--------|
| `learning_rate` | 学习率 | 1e-3 ~ 3e-4 |
| `num_learning_epochs` | 每次收集数据后训练几轮 | 5 |
| `num_mini_batches` | 数据分成几个 mini-batch | 4 |
| `clip_param` | PPO clip 范围 | 0.2 |
| `gamma` | 折扣因子 | 0.99 |
| `lam` | GAE lambda | 0.95 |
| `entropy_coef` | 熵正则化系数 | 0.01 |
| `max_grad_norm` | 梯度裁剪 | 1.0 |

### 5.2 Actor-Critic 网络

```
观测向量 (obs_dim)
    │
    ├─→ Actor 网络 → 动作均值 (action_dim)
    │   [MLP: 256→256→256]     → 动作方差 (可学习)
    │                           → 采样动作
    │
    └─→ Critic 网络 → 状态价值 (1)
        [MLP: 256→256→256]
```

### 5.3 Domain Randomization

训练时随机化仿真参数，提高策略的鲁棒性：

```python
# 在环境配置中添加 randomization
class MyHumanoidRandomizeCfg:
    # 物理参数随机化
    mass_randomization = (+/- 15%)      # 各 link 质量
    friction_randomization = (0.5, 1.5) # 地面摩擦系数
    restitution_randomization = (0, 0.5) # 弹性系数

    # 控制参数随机化
    stiffness_randomization = (+/- 20%) # PD 控制刚度
    damping_randomization = (+/- 20%)   # PD 控制阻尼

    # 外部扰动
    push_force_range = (-50, 50)  # 随机推力 (N)
    push_interval = (5, 15)       # 推力间隔 (s)

    # 传感器噪声
    imu_noise_std = 0.05          # IMU 噪声标准差
    joint_pos_noise = 0.01        # 关节位置噪声 (rad)

    # 延迟
    action_delay = (0, 2)         # 动作延迟 (control steps)
```

### 5.4 Curriculum Learning

```
阶段 1: 平坦地形，低速命令 (0~0.5 m/s)
  ↓ (达到一定 reward 阈值)
阶段 2: 轻微起伏地形，中速命令 (0~1.0 m/s)
  ↓
阶段 3: 斜坡 + 台阶，全速命令 (0~1.5 m/s)
  ↓
阶段 4: 复杂地形 + 外部扰动 + 大 DR
```

## 7. 训练注意事项

### 7.1 常见训练失败与排查

| 症状 | 可能原因 | 解决方案 |
|------|---------|---------|
| reward 始终为 0 | 机器人立即摔倒 | 检查初始姿态、增大存活奖励 |
| reward 剧烈震荡 | 学习率过大 | 降低 learning_rate |
| 原地不动（高 reward） | 奖励函数有漏洞（站着不动也有高奖励） | 增加速度跟踪奖励权重 |
| 滑步（脚在地上滑动） | 摩擦力不足或接触模型问题 | 增大摩擦系数、检查碰撞设置 |
| 单脚跳 | feet_air_time 设计有问题 | 确保左右脚分别计算空中时间 |
| 训练极慢 | num_envs 太少 | 增大并行环境数 |

### 7.2 超参数调优建议

1. **先用小环境数 (256-512) 快速迭代**，确认奖励设计合理
2. **再用大环境数 (4096) 正式训练**
3. **每次只改一个变量**，对比效果
4. **保存多个 checkpoint**，方便回溯
5. **使用 TensorBoard 监控**所有指标

### 7.3 仿真参数

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| `physics_dt` | 1/200 ~ 1/500 | 越小越精确，但越慢 |
| `control_dt` | 1/50 ~ 1/100 | 需与实机控制频率匹配 |
| `solver_iterations` | 4~8 | 人形机器人可能需要更高值 |
| `max_gpu_contact_pairs` | 2^23 | 默认值，通常够用 |
